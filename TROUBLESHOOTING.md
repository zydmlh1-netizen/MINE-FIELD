# 🔧 دليل حل المشاكل - حقل الألغام

## ❌ خطأ "لم يُعثر على الغرفة"

عندما تحصل على رسالة الخطأ هذه عند محاولة الانضمام إلى غرفة:
```
❌ لم يُعثر على الغرفة.
تحقق من:
✓ الكود صحيح (6 أحرف)
✓ Firebase مهيأ
✓ الخصم في الغرفة
```

### 🔍 خطوات حل المشكلة:

#### 1️⃣ تحقق من كود الغرفة
- ✅ الكود يجب أن يكون **6 أحرف** تماماً
- ✅ تأكد من أن الشخص الآخر قد **نسخ الكود بشكل صحيح**
- ✅ تحقق من عدم وجود **مسافات إضافية**

#### 2️⃣ تأكد من وجود Firebase

**اختبر اتصال Firebase:**
1. افتح أدوات المتطور (F12 في المتصفح)
2. اذهب إلى تبويب **Console**
3. حاول الانضمام إلى غرفة
4. ابحث عن الرسائل التشخيصية:
   - ✅ `✅ Firebase متصل:` = اتصال سليم
   - ❌ `❌ Firebase غير متصل` = مشكلة اتصال

#### 3️⃣ تحقق من بيانات Firebase

**المشكلة الأكثر شيوعاً:** استخدام بيانات Firebase الوهمية

اتبع هذه الخطوات:
1. اذهب إلى [Firebase Console](https://console.firebase.google.com)
2. اختر مشروعك
3. انقر على "Realtime Database"
4. تأكد من وجود بيانات في القاعدة (يجب أن ترى شجرة البيانات)

**إذا كانت القاعدة فارغة:**
- الكود يستخدم بيانات وهمية
- اتبع تعليمات SETUP.md لتحديث `firebase-config.js`

#### 4️⃣ تحقق من قواعد الأمان في Firebase

في Firebase Console:
1. انقر على "Realtime Database"
2. انقر على تبويب **"Rules"**
3. تأكد من أن القواعد تسمح بالقراءة والكتابة:

```json
{
  "rules": {
    "rooms": {
      "$roomCode": {
        ".read": true,
        ".write": true,
        "messages": {
          "$messageId": {
            ".read": true,
            ".write": true
          }
        }
      }
    }
  }
}
```

📝 **ملاحظة:** هذه قواعد اختبار فقط. للإنتاج، استخدم قواعد أمان أقوى.

#### 5️⃣ تحقق من تسلسل الاتصال

تأكد من أن **المضيف (Host) يُنشئ الغرفة أولاً:**

1. الشخص الأول: ✅ انقر "إنشاء غرفة" (ينتظر الخصم)
2. الشخص الثاني: ✅ انقر "انضمام" وأدخل الكود
3. **الترتيب مهم جداً!**

#### 6️⃣ اختبر بدون Firebase (وضع محلي)

إذا استمرت المشكلة:
1. افتح **نافذتي متصفح** على **نفس الجهاز**
2. اختبر اللعب المحلي بينهما
3. إذا عمل محلياً، فالمشكلة في Firebase

## 🌐 اختبار الاتصال بـ Firebase

### فتح Console في المتصفح:
- **Chrome**: اضغط `F12` → انقر "Console"
- **Firefox**: اضغط `F12` → انقر "Console"
- **Safari**: اضغط `Cmd+Option+I` → انقر "Console"

### رسائل التشخيص المهمة:

| الرسالة | المعنى | الحل |
|--------|--------|------|
| `✅ Firebase متصل` | اتصال سليم | ✅ لا توجد مشكلة |
| `❌ Firebase غير متصل` | لا يوجد اتصال | ✅ تحقق من الإنترنت والبيانات |
| `❌ الغرفة غير موجودة` | الكود خاطئ أو المضيف لم يُنشئها | ✅ تحقق من الكود والترتيب |
| `✅ تم التحقق من وجود الغرفة` | الاتصال سليم | ✅ جاهز للعب |

## 📝 قائمة التحقق النهائية

- [ ] استخدمت بيانات Firebase الحقيقية (ليست وهمية)
- [ ] المضيف أنشأ الغرفة أولاً
- [ ] الضيف استخدم الكود الصحيح (6 أحرف)
- [ ] كلاهما لديهما اتصال إنترنت
- [ ] Firebase Console تظهر البيانات
- [ ] قواعد الأمان تسمح بالقراءة والكتابة

---

## 💡 نصائح إضافية

### إذا استمرت المشاكل:
1. **أعد تحميل الصفحة** في كلا المتصفحين
2. **امسح ذاكرة التخزين المؤقت** (Clear Cache)
3. **استخدم متصفح آخر** (Chrome/Firefox)
4. **اختبر بنافذة حاضرة** (Incognito Window)
5. **تحقق من جدار الحماية** قد يحجب الوصول

### للتطوير والاختبار:
- استخدم **Firebase Emulator** لاختبار محلي
- افتح **DevTools Console** لرؤية رسائل التشخيص
- استخدم **Network Tab** لمراقبة طلبات Firebase

---

## 📞 إذا استمرت المشكلة

تحقق من:
1. رسائل Console في متصفح كلا اللاعبين
2. بيانات Realtime Database في Firebase Console
3. قواعد الأمان (Firebase Rules)
4. اتصال الإنترنت على كلا الجهازين
